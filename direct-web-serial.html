direct web serial grid notes

A practical guide to talking to a monome grid directly from a browser, without serialosc, OSC, Node, npm, or a bridge process.

back to instruments

what this is

Modern monome grids are USB serial devices. The usual desktop path is grid -> serialosc -> OSC client. Web Serial lets Chrome/Edge open the USB serial port directly from a local or HTTPS page, so a web app can read grid key packets and write LED packets by using the raw serial protocol.

This is lower level than serialosc. You are responsible for packet parsing, LED packing, port ownership, and browser permissions.

requirements

  • Chrome, Chromium, Edge, or another browser with the Web Serial API.
  • A secure context: http://localhost, http://127.0.0.1, or HTTPS.
  • serialosc stopped, or at least not holding the grid serial port.
  • A user gesture for connection. navigator.serial.requestPort() must be called from a click or similar event.
brew services stop serialosc
python3 -m http.server 8000
# open http://localhost:8000/ in Chrome or Edge

# after use, restore normal monome app access
brew services start serialosc

protocol pieces used here

message bytes meaning
query size 0x05 Ask the grid to report its size.
size report 0x03, width, height Returned by the grid after a size query.
key up 0x20, x, y A key was released.
key down 0x21, x, y A key was pressed.
all LEDs off 0x12 Clear grid LEDs.
varibright map 0x1A, x, y, 32 packed bytes Write one 8x8 LED tile with 0-15 brightness values.

Coordinates are zero-indexed. On a 16x8 grid, x is 0..15 and y is 0..7.

tiny library

The smallest reusable piece in this folder is grid-serial.js. It is a plain browser ES module: no build step, no npm package, no server code. Import it, create a grid, provide callbacks, then draw 0-15 LED levels.

import { MonomeGridSerial, makeGridLevels } from "./grid-serial.js";

const levels = makeGridLevels(16, 8);

const grid = new MonomeGridSerial({
  onKey(x, y, down) {
    levels[y][x] = down ? 15 : 0;
    grid.draw(levels);
  },
  onSize(cols, rows) {
    console.log(`grid size: ${cols} x ${rows}`);
  }
});

document.querySelector("button").addEventListener("click", () => {
  grid.connect();
});

The library intentionally stays small. It handles Web Serial connection, packet buffering, size reports, key callbacks, LED clearing, and 8x8 varibright packing. Your page still owns the instrument state and display.

<script type="module">
  import { MonomeGridSerial, makeGridLevels } from "./grid-serial.js";

  const mirror = document.querySelector("#mirror");
  const log = document.querySelector("#log");
  let levels = makeGridLevels(16, 8);

  const grid = new MonomeGridSerial({
    onSize(cols, rows) {
      levels = makeGridLevels(cols, rows);
      mirror.style.gridTemplateColumns = `repeat(${cols}, 18px)`;
      renderMirror();
      log.textContent = `connected: ${cols} x ${rows}`;
    },
    onKey(x, y, down) {
      levels[y][x] = down ? 15 : 0;
      renderMirror();
      grid.draw(levels);
      log.textContent = `key ${x}, ${y}: ${down ? "down" : "up"}`;
    },
    onStatus(message) {
      log.textContent = message;
    }
  });

  function renderMirror() {
    mirror.replaceChildren();
    for (let y = 0; y < grid.rows; y += 1) {
      for (let x = 0; x < grid.cols; x += 1) {
        const key = document.createElement("div");
        key.className = levels[y]?.[x] ? "key on" : "key";
        mirror.appendChild(key);
      }
    }
  }

  document.querySelector("#connect").addEventListener("click", () => {
    grid.connect().catch((error) => {
      log.textContent = error.message;
    });
  });
</script>

no-library reference

This complete HTML page shows the lower-level version behind the library: connect to a grid, mirror key presses in memory, and light pressed keys. It assumes a 16x8 layout until the grid reports size.

<!doctype html>
<button id="connect">connect grid</button>
<pre id="log"></pre>

<script>
const MSG = {
  SIZE_REQUEST: 0x05,
  SIZE_REPORT: 0x03,
  LED_ALL_OFF: 0x12,
  LED_LEVEL_MAP: 0x1A,
  KEY_UP: 0x20,
  KEY_DOWN: 0x21
};

let port, reader, writer;
let cols = 16;
let rows = 8;
let pending = [];
let levels = Array.from({ length: rows }, () => Array(cols).fill(0));

const log = (text) => {
  document.querySelector("#log").textContent = text;
};

async function write(bytes) {
  if (writer) await writer.write(new Uint8Array(bytes));
}

function resizeGrid(c, r) {
  cols = c || 16;
  rows = r || 8;
  levels = Array.from({ length: rows }, () => Array(cols).fill(0));
}

function packetLength(first) {
  if (first === MSG.SIZE_REPORT || first === MSG.KEY_UP || first === MSG.KEY_DOWN) return 3;
  return 1;
}

async function consume(bytes) {
  pending.push(...bytes);
  while (pending.length) {
    const len = packetLength(pending[0]);
    if (pending.length < len) return;
    await handlePacket(pending.splice(0, len));
  }
}

async function handlePacket(packet) {
  const [type, a, b] = packet;

  if (type === MSG.SIZE_REPORT) {
    resizeGrid(a, b);
    log(`grid size: ${cols} x ${rows}`);
    await draw();
    return;
  }

  if (type !== MSG.KEY_UP && type !== MSG.KEY_DOWN) return;
  if (a >= cols || b >= rows) return;

  const pressed = type === MSG.KEY_DOWN;
  levels[b][a] = pressed ? 15 : 0;
  log(`key ${a}, ${b}: ${pressed ? "down" : "up"}`);
  await draw();
}

function pack8x8(x0, y0) {
  const packed = [];
  for (let i = 0; i < 32; i += 1) {
    const aIndex = i * 2;
    const bIndex = aIndex + 1;
    const ax = x0 + (aIndex % 8);
    const ay = y0 + Math.floor(aIndex / 8);
    const bx = x0 + (bIndex % 8);
    const by = y0 + Math.floor(bIndex / 8);
    const a = Math.max(0, Math.min(15, Math.round(levels[ay]?.[ax] ?? 0)));
    const b = Math.max(0, Math.min(15, Math.round(levels[by]?.[bx] ?? 0)));
    packed.push((a << 4) | b);
  }
  return packed;
}

async function draw() {
  for (let y = 0; y < rows; y += 8) {
    for (let x = 0; x < cols; x += 8) {
      await write([MSG.LED_LEVEL_MAP, x, y, ...pack8x8(x, y)]);
    }
  }
}

async function readLoop() {
  while (port?.readable) {
    reader = port.readable.getReader();
    try {
      while (true) {
        const { value, done } = await reader.read();
        if (done) break;
        if (value) await consume(value);
      }
    } finally {
      reader.releaseLock();
    }
  }
}

document.querySelector("#connect").addEventListener("click", async () => {
  if (!("serial" in navigator)) {
    log("Web Serial is unavailable. Use Chrome or Edge.");
    return;
  }

  port = await navigator.serial.requestPort();
  await port.open({ baudRate: 115200, dataBits: 8, stopBits: 1, parity: "none", flowControl: "none" });
  writer = port.writable.getWriter();

  readLoop();
  await write([MSG.LED_ALL_OFF]);
  await write([MSG.SIZE_REQUEST]);
  log("connected");
});
</script>

led packing

The varibright LED message writes one 8x8 tile at a time. There are 64 LED levels, each 0-15. Two 4-bit levels are packed into one byte, so the payload is 32 bytes.

// first cell goes in the high nibble, second in the low nibble
const packedByte = (levelA << 4) | levelB;

For a 16x8 grid, send two maps: one at x=0,y=0 and one at x=8,y=0. For 16x16, send four maps.

Physical LED response does not necessarily feel linear. In this project the app computes 0-15 frames, then applies a hardware brightness curve before sending to the grid so only active hits are fully bright.

important gotchas

  • serialosc conflict: if serialosc owns the port, the browser may fail to open it. Stop serialosc first.
  • serialosc restart: after using a direct Web Serial page, run brew services start serialosc if you want regular monome apps to use the grid again.
  • browser support: Web Serial is Chromium-only in practice. Safari and Firefox are not suitable for this.
  • permission: the port picker only opens from a user gesture. You cannot auto-connect on page load.
  • secure context: use localhost or HTTPS. Opening the file directly with file:// is not enough for Web Serial.
  • packet boundaries: serial reads may split or combine packets. Always buffer bytes and parse complete packets.
  • LED write rate: do not fire unlimited LED frames. Coalesce redraws if your app animates quickly.
  • disconnect cleanup: release reader/writer locks and clear LEDs before closing the port.
  • device choice: the browser picker shows serial devices. Pick the monome grid, not another USB serial device.

a reusable shape

For a real app, keep the grid transport separate from the instrument logic. The transport should expose only a few operations.

const grid = {
  cols: 16,
  rows: 8,
  onKey: (x, y, down) => {},
  connect: async () => {},
  draw: async (levels) => {},
  clear: async () => {}
};

// Your instrument can then be just:
function key(x, y, down) {
  if (!down) return;
  pattern[y][x] = !pattern[y][x];
  grid.draw(renderPattern(pattern));
}

This makes it easy to build alternate renderers: physical grid, browser mirror, debugging console, or a simulated grid for development without hardware.
